# 本地缺陷修复日志（简版）

- 影响模块：平衡、结算、回合战斗、测试。

### 2) 世界 Boss 状态获取失败
- 修复了旧表结构缺字段导致的状态获取失败。
- 增加首次写入并发保护，避免首访冲突报错。
- 修复历史异常 Boss 血量字段数据。
- 影响模块：活动与世界 Boss 服务。

- 影响模块：宗门加成、状态面板、战斗。

- 影响模块：抽奖服务与 Telegram 抽奖交互。

- 突破不满足试炼条件时，明确提示缺失任务与剩余进度。

- 影响模块：配置、活动服务、Boss 交互文案。

### 10) 修炼/突破审查文档一问题修复（并发与提示一致性）
- 修炼开始/结束改为原子流程，修复并发重复开始、重复结算风险。
- 修炼结束与日奖励发放改为单事务结算，避免并发重复发奖。
- 境界试炼进度更新改为并发安全（行锁 + 原子更新），修复丢进度问题。
- 试炼记录创建增加并发冲突兜底（`ON CONFLICT DO NOTHING`）。
- 试炼进度更新异常不再静默吞掉，增加错误日志与埋点。
- 突破失败事件日志改为 `success=False`，并区分业务成功语义字段。
- 突破预览新增后端接口，Bot 优先使用服务端预览，降低前后端规则漂移。
- 修炼状态面板增加成功态校验，失败时明确提示而不是误显示 0 收益。
- `GET /api/realm-trial/<user_id>` 改为纯读取，不再触发写入创建。
- 影响模块：`cultivation` 路由、`realm_trials` 服务、`settlement`、`settlement_extra`、Telegram Bot。

### 11) 战斗/狩猎审查文档二问题修复（稳定性与交互完整性）
- 回合战属性链路修复：宗门加成与虚弱减益不再被覆盖或绕过。
- 回合会话并发保护增强：会话表统一加锁访问，避免并发读写异常。
- 会话 ID 改为高熵 UUID，并增加冲突检测重试。
- 回合 action 增加 `request_id` 幂等去重，避免重复点击推进多回合。
- 战斗路由布尔参数改为严格解析，修复 `"false"` 被误判为开启。
- Telegram 秘境补齐 `needs_choice` 选项流，并接入 choice 回调。
- `/hunt <monster_id>` 统一走回合制链路，减少按钮/命令双引擎分叉。
- `kernel_shadow` 默认关闭；失败原因“五行被克”文案匹配修正。

### 12) 经济系统审查文档四问题修复（并发/幂等/配置安全）
- 转化与炼丹扣材改为并发安全扣减（数量守卫 + `rowcount` 校验），修复并发超扣风险。
- 抽卡保底改为事务内读取与原子 upsert，补充并发冲突兜底返回，避免首抽并发异常。
- 货币兑换、普通锻造、定向锻造、抽卡统一接入 `request_id` 幂等；对应路由与 Telegram 回调/命令已透传。
- 修复 `force_paid` 严格布尔解析，避免 `"false"` 被误判为 `True`。
- 定向锻造补齐全局开关校验（`forge.enabled`）。
- 抽卡热路径移除重复 DDL，改为单次初始化。
- 转化配置增加硬校验（`base_copper`、`cost_mult`、`max_batch`、`success_rate`、`fail_output_mult`）。
- 转化超批量请求改为显式报错，不再静默截断。
- 货币总览 `wallet` 与可见性策略一致，未解锁高阶货币脱敏。
- 转化/炼丹经济埋点 `success` 改为业务成功语义。

### 13) 社交/宗门/任务审查文档五问题修复（原子性与输入校验）
- 论道接受流程改为事务内统一抛错回滚，修复“状态已改但奖励未完整发放”的风险。
- 悬赏系统补齐托管字段与一次性 schema 兜底；发布时预扣奖励，提交时优先走托管结算。
- 悬赏提交流程改为事务内失败统一回滚；仅允许材料类道具参与悬赏流转。
- 宗门别院审批 `approve` 改为严格布尔解析，修复字符串误判同意的问题。
- 宗门捐献补齐整数参数校验，非法输入返回 400，不再触发 500。
- 别院申请前置条件与审批条件统一：仅主宗直系成员可发起创建申请。
- 成就列表改为一次建表 + 一次查询已领集合，移除重复 DDL 与 N+1 查询。
- 成就领取增加账户更新 `rowcount` 校验，避免“标记已领但未到账”。
- Telegram 的 `sect_menu / quests / quest_claim` 在账号映射失败时补齐可见错误提示。

### 14) 世界Boss/活动审查文档三问题修复（并发与健壮性）
- 世界 Boss 攻击改为行锁流程（`FOR UPDATE`），修复并发丢伤害与重复击杀结算风险。
- 旧世界 Boss 表结构补齐去重与唯一索引兜底，降低历史库脏数据影响。
- 活动积分兑换改为并发安全结算（锁定积分与兑换计数，原子扣减/限次）。
- 活动兑换数量参数补齐路由层与服务层整数校验，非法输入统一 400。
- 活动周周期键改为自然周（ISO week）语义，不再使用简单日序号整除。
- 世界 Boss 配置 `max_hp` 增加正数钳制与安全整型转换，防止异常配置导致玩法失效。
- 事件引擎条件判断补齐类型容错（境界、道途、随机概率）。
- Telegram 活动领取在修炼中拦截文案改为“领取活动奖励”。
